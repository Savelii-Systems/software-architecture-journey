# ADR-2026-001: Выбор языка Go для обучения разработке

## Status
Accepted (2026-01-15)

## Context
Необходимо выбрать первый язык программирования для понимания mindset разработчика и понимания кода + cross-skilling.

## Decision
Выбрать Go (Golang).

## Rationale
### Критерии выбора:
1. **Цели обучения:**
   - Понимание статической типизации
   - Опыт работы с компилируемым языком
   - Практика конкурентного программирования

2. **Сравнение альтернатив:**
   | Язык | Плюсы | Минусы | Итог |
   |------|-------|--------|------|
   | **Go** | Простая конкурентность, стабильный API, растущий рынок, используется на проекте | Меньше вакансий чем Java/Python | ✅ Выбор |
   | Python | Легко учить, много библиотек | Динамическая типизация скрывает ошибки | ❌ Не подходит |
   | Java | Распространён в enterprise | Сложный для начала | ❌ Слишком сложно |

3. **Бизнес-контекст:**
   - Go используется в микросервисах (мой целевой домен)
   - Синтаксис похож на C-подобные языки
   - Сообщество растёт

## Consequences
### Положительные:
1. Получу опыт современного backend языка
2. Умение работать с горутинами и каналами
3. Поняю компиляцию и линковку

### Отрицательные:
1. Кривая обучения круче, чем у Python
2. Ограниченное применение вне backend
3. Меньше материалов для аналитиков

### Технический долг:
- Нужно будет изучать основы компиляции
- Придётся разбираться с указателями
